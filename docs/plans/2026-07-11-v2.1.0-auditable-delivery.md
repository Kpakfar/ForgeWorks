# v2.1.0 "Auditable delivery" — implementation plan

Source: 2026-07-11 field audit of v2.0.0 in the Heurema capstone (git + session-transcript
forensics by Claude Fable 5, two-round adversarial convergence with Codex 5.6 Sol run
read-only inside this repo and the capstone). Status: **plan approved by both models,
not yet approved by the maintainer, not yet implemented.**

## Field findings this fixes (v2.0.0 in production, slices [053]/[056])

- `@security-reviewer` never ran, even for a slice publishing LLM-authored content to
  public GitHub Pages; "escape hatch" roster framing + token discipline beat the
  prose mandate. `docs/SECURITY.md` went stale for the new surface.
- [056] Red phase was retroactive (implementer wrote tests+code together, stashed the
  implementation to prove failure); tests+impl landed in one commit — git cannot prove TDD.
- A user lean-mode directive (`decide-don't-enumerate` + no-pause Stop hook) suppressed
  exploratory grilling; the harness let comms compression also compress interrogation.
- A bot-identity prompt change ([058]) shipped memo-less via the trivial-task bypass.
- Main-loop fix-rounds and a "lean re-review in main context" happened outside the rules.
- `@tech-debt` cadence never fired; a 240-line test file passed review despite the 200 cap.
- What worked and must not regress: design-memo hard gate (both slices user-approved
  before code), real non-ceremonial tests, ~3x leaner transcripts.

## v2.1.0 work items (converged final list)

1. **Canonical delivery contract + durable ship record.**
   New FW-BLOCK `delivery-evidence v2.1.0` in `templates/core/AGENTS.md`; bump
   `backlog-discipline`, `task-specific-documents`, `global-documents`. New
   `docs/ships/README.md` (schema); update `task-template.md`, `backlog.md`,
   `structure.txt`. Schema per shipped slice: task class; memo-approved-before-code;
   Red proof; reviewers run; security surface/disposition; Green proof;
   `TDD audit: strong|weak` (+reason); `Evidence origin: native|imported`
   (+branch/PR ref); total review/fix rounds. Ship records are permanent files, not
   commit-message trailers (task.md gets reset; squashes hide churn).

2. **Agent-neutral audit CLI + local commit guard.**
   New `templates/core/.claude/hooks/slice-audit.sh` — a plain CLI, invoked by BOTH a
   `PreToolUse(Bash)` hook (fires only on `git commit`; acts only when the staged diff
   moves a backlog row to Shipped; validates ship-record presence + headings, no git
   archaeology) AND CI. Merge hook into `.claude/settings.json`; bump `quality-gate`.

3. **Independent CI audit job.**
   `templates/core/.github/workflows/qa.yml`: `fetch-depth: 0`; validate changed ship
   records; check native memo->Red->Green commit ordering; `Evidence origin: imported`
   permits parked-branch/prior-session evidence and shifts provenance to the reviewer.
   New `.github/scripts/slice_audit_test.sh` fixtures in root CI. Runs even when the
   reviewer subagent was skipped (audit independent of subagent invocation).

4. **Mandatory vs optional roster + ONE canonical security trigger.**
   Bump `agent-roster`, `recurring-reviews`, `security-discipline`, conditional
   `ai-discipline`. Canonical sentence (quoted everywhere, single source in
   `delivery-evidence`): mandatory `@security-reviewer` for any slice adding
   *external input, dependence on untrusted generated output, public publishing, auth,
   tools, or persistence of untrusted content*; `ai-discipline` maps "LLM output a
   downstream depends on" to untrusted generated output. Kill "escape hatch" framing
   for the mandatory tier (also in README/SKILL.md meta text — they currently teach
   the bypass). Update `security-reviewer.md`, `code-reviewer.md`, `docs/SECURITY.md`.

5. **Threat-model freshness gate.**
   Bump `planning-discipline`, `security-discipline`. Security-triggered slices must
   land a `docs/SECURITY.md` delta or a written `Security doc delta: none, because ...`
   in the memo before security review; reviewer blocks ship on stale/unexplained
   threat model. Memo template (`docs/designs/README.md`) gains `Security surface:`
   field.

6. **Memo floor + evidence classification.**
   Bump `planning-discipline`, `investigation-discipline`. Floor headings every
   non-trivial memo must answer: core journey, concrete examples, options considered,
   riskiest assumption + probe result, unhappy/missing paths, attacker story,
   non-goals, boundary data shapes, AC-to-test mapping, security disposition.
   Lean/no-pause/token-saving directives may compress wording but cannot remove a
   heading or start code pre-approval. One-page TARGET stays; completeness wins on
   conflict. Retroactive Red (stash-and-rerun) = `TDD audit: weak`. Native flow
   commits memo -> Red -> Green separately.

7. **Scoped main-loop fix-rounds + behavioral triviality.**
   Bump `token-discipline`, `exceptional-cases`. Legal: after the single full review,
   main context may fix named findings directly (scoped, affected checks only,
   increment total fix-round count; expansion/design change/2 failures -> back to memo).
   Trivial-task bypass excludes anything touching runtime behavior, identity,
   permissions, safety policy, or user-visible output — prompt/persona text is never
   trivial (fixes the [058] class). Fix README "sub-1h skips ceremony" contradiction.

8. **Generator, reconciler, release, live validation.**
   `init-project/SKILL.md` Phase 4 (AI-block wording) + Phase 5 (check
   `docs/ships/README.md`, `slice-audit.sh`); `upgrade-project/SKILL.md` 3-A (new
   files + settings/hook merge), 3-B (~12 bumped blocks side-by-side), 3-C (CI audit
   job delta); bump `VERSION`, bootstrap REF, one-liners. Validate: root CI + 3
   profile renders + audit fixtures (native and imported git histories) + throwaway
   AI/non-AI bootstraps green on first run. **Acceptance: run released
   /upgrade-project on the Heurema capstone** and inspect grafts, imported parked-Red
   handling, hook merge, idempotent second run.

## Resolved disagreements (for the record)

- Per-finding fix-round counts: dropped (Claude) — total count only, circuit breaker
  stays reviewer-enforced prose.
- "One page or less": kept as target, not dropped (Claude); completeness wins (Codex).
- History archaeology in the commit hook: rejected (Claude) — hook checks presence +
  headings only; ordering evidence lives in CI + reviewer with the `imported` escape.
- Line caps / tech-debt cadence in v2.1.0: deferred to v2.1.1 (both, round 2).
- Independent security reviewer for publish/LLM-output surfaces: mandatory, folding
  into code-reviewer is not a substitute (Codex, upheld).
