# Backlog

Each row is a vertical slice that ships a demoable step end-to-end. See `AGENTS.md` `<backlog-discipline>` for the rule.

Rows move: `proposals-ideas.md` -> `backlog.md` (Active) -> `current-task/task.md` -> `backlog.md` (Shipped).

## Format

```
### [ID] Slice title -- the observable outcome when shipped
- Why: one-sentence motivation tied to the product vision
- Acceptance criteria:
  - [ ] criterion 1 (user-observable, end-to-end)
  - [ ] criterion 2
- Estimated effort: S / M / L (under 1h / half-day / multi-day)
- Dependencies: list other slices this blocks on
- Links: docs, tickets, PRs
```
